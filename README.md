# Parmanu Seniors Health

Website for DAE pensioners and CHSS beneficiaries, managed by Dr. Venu Babu Pentyala.

**Live site:** https://paramanuseniorshealth.org

---

## Stack Overview

| Layer | Technology |
|-------|-----------|
| Static site generator | Hugo 0.145.0 |
| Theme | Custom fork of `hugo-bootstrap-theme` (Bootstrap 5) |
| CMS | Decap CMS (formerly Netlify CMS) |
| CMS auth backend | Cloudflare Workers |
| Hosting | GitHub Pages |
| CI/CD | GitHub Actions |
| Analytics | Google Analytics 4 + Google Tag Manager |

## Hugo (Static Site Generator)

The site is built with [Hugo](https://gohugo.io/) using a custom fork of the Bootstrap 5 theme at `themes/hugo-bootstrap-theme` (git submodule from `revanthp/hugo-bootstrap-theme`).

Configuration lives in `config/_default/`:

| File | Purpose |
|------|---------|
| `config.toml` | Base URL, Google Analytics ID (`G-BGSPDZEPPY`), robots.txt, timezone |
| `params.toml` | Bootstrap variant, logo, Open Graph / Twitter Card metadata |
| `markup.toml` | Goldmark markdown extensions (tables, footnotes, typography) |
| `languages.toml` | English (`en-US`) |
| `menus.toml` | Top-nav and footer links |

Build command: `hugo --gc --minify`

### Decap CMS

The CMS is accessible at `/admin/` and is powered by [Decap CMS](https://decapCMS.org/).

- **Config:** `static/admin/config.yml`
- **Backend:** GitHub (repo `revanthp/parmanuseniorhealth.github.io`, branch `main`)
- **Media folder:** `static/files/`
- **Editorial workflow:** enabled (PRs created per save, squash-merged on publish)
- **Collections managed via CMS:** Pages, General Services, Health Matters, Pension, Telephone Numbers

Editors log in with their GitHub account. On save, Decap opens a pull request against `main`; publishing merges it, which triggers the CI/CD build.

## Authentication (CMS Login)

Decap CMS uses GitHub OAuth for editor login via the **implicit flow**:

- **Auth endpoint:** `https://decap.paramanuseniorshealth.org/auth`
- This endpoint is a **Cloudflare Worker** that handles the OAuth callback with GitHub and returns a token to the CMS UI.
- No site-level authentication exists — the public site is fully open.

---

## Cloudflare

Cloudflare serves two roles:

1. **CMS OAuth Worker** — The subdomain `decap.paramanuseniorshealth.org` runs a Cloudflare Worker that acts as the OAuth backend for Decap CMS. This is what allows editors to log in with GitHub without needing a traditional server.
2. **DNS / CDN** (if configured) — Cloudflare may sit in front of the GitHub Pages origin for DNS management, DDoS protection, and caching.

The `wrangler` CLI (`^4.14.4` in `package.json`) is used to deploy and manage the Cloudflare Worker.

## CI/CD (GitHub Actions → GitHub Pages)

Workflow: `.github/workflows/hugo.yml`

**Trigger:** push to `main` branch, or manual `workflow_dispatch`.

**Steps:**
1. Check out repo with submodules (theme)
2. Cache Hugo modules (`/tmp/hugo_cache`)
3. Install dependencies via `.github/actions/install-dependencies/` (Hugo CLI 0.145.0, Dart Sass)
4. Install Node dependencies (`npm ci`) for Wrangler tooling
5. Run `hugo --gc --minify` with `HUGO_ENVIRONMENT=production`
6. Deploy the `public/` output to GitHub Pages via `actions/deploy-pages@v4`

**Permissions used:** `contents: read`, `pages: write`, `id-token: write`

The full pipeline — CMS edit → PR → merge → build → deploy — is automated end to end.

## Accessibility

The site is designed for senior citizens. Rules are documented in `.rules`:

- Minimum font size 16 px; font-resizer widget in the navbar
- WCAG AA contrast ratios
- Minimum touch targets 40 px
- Plain language, simple navigation structure
- Mobile-responsive layout
